上一篇看了 P4 程序的整体结构，这一篇深入理解 Header——P4 中最基础的概念。

---

## 一、什么是 Header

Header 是 P4 对**网络报头**的抽象。每个网络协议（以太网、IP、TCP 等）都有自己的报头格式，Header 就是用代码描述这些格式。

可以类比为：
- **C 语言**：`struct` 定义数据结构
- **P4**：`header` 定义报文结构

---

## 二、定义一个 Header

以太网帧的报头：

```p4
header Ethernet_h {
    bit<48> dstAddr;    // 目的 MAC 地址
    bit<48> srcAddr;    // 源 MAC 地址
    bit<16> etherType;  // 协议类型
}
```

IPv4 报头（简化版）：

```p4
header IPv4_h {
    bit<4>  version;    // 版本
    bit<4>  ihl;        // 头部长度
    bit<8>  diffserv;   // 服务类型
    bit<16> totalLen;   // 总长度
    bit<32> srcAddr;    // 源 IP
    bit<32> dstAddr;    // 目的 IP
    bit<8>  protocol;   // 协议号
}
```

## 三、基本类型

### 3.1 bit\<W\>：无符号整数

最常用的类型，W 表示位宽：

```p4
bit<8>   port;      // 8 位，0~255
bit<16>  etherType; // 16 位，0~65535
bit<48>  macAddr;   // 48 位，MAC 地址
bit<32>  ipv4Addr;  // 32 位，IPv4 地址
```

### 3.2 int\<W\>：有符号整数

网络报文处理中很少用到，但可以定义：

```p4
int<8>  temperature;  // -128 ~ 127
int<16> offset;       // -32768 ~ 32767
```

### 3.3 bool：布尔值

```p4
bool flag = true;
```

### 3.4 格式：`<宽度>w<值>`

P4 的字面量可以显式指定宽度：

```p4
32w255           // bit<32>，值 255
32w0xFF          // 同上，十六进制
8w0b10101010     // bit<8>，值 170
16w0x0800        // bit<16>，值 2048（IPv4 的 etherType）
```

---

## 四、Header Stack：处理重复报头

有些协议的报头会出现多次（如 VLAN 标签），用 Header Stack 处理：

```p4
header_stack<2, Ethernet_h> vlanTags;  // 最多 2 层 VLAN
```

访问方式：
```p4
vlanTags[0].vid  // 第一层 VLAN 的 VID
vlanTags[1].vid  // 第二层 VLAN 的 VID
```

---

## 五、Header Union：处理可选报头

不同协议的报头不能同时存在，用 Union 表示互斥关系：

```p4
union L3_h {
    IPv4_h ipv4;
    IPv6_h ipv6;
}
```

---

## 六、Headers 结构

把所有报头组合成一个结构：

```p4
struct Headers {
    Ethernet_h      ethernet;
    vlan_tag[2]     vlan;      // VLAN 标签栈
    L3_h            l3;        // L3 协议（IPv4 或 IPv6）
    TCP_h           tcp;
    UDP_h           udp;
}
```

---

## 七、isValid()：检查报头是否有效

Parser 解析后，只有被 extract 的 header 才是"有效的"：

```p4
if (hdr.ipv4.isValid()) {
    // IPv4 头存在，可以处理
}
```

---

## 八、小结

| 概念 | 说明 |
|------|------|
| `header` | 定义报文结构 |
| `bit<W>` | W 位无符号整数 |
| `header_stack` | 处理重复报头 |
| `union` | 处理互斥报头 |
| `isValid()` | 检查报头是否有效 |
