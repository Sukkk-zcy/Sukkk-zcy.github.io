前面的实验都是静态配置表项。实际使用中，表项由控制平面动态下发——这就是 P4Runtime 的作用。

---

## 一、P4Runtime 是什么

P4Runtime 是一个标准的**控制平面-数据平面通信协议**，用于：

1. 下发表项（Match-Action 表）
2. 收发报文（Packet-in/Packet-out）
3. 读取统计信息（Counter 等）

---

## 二、工作流程

```
1. 控制器读取 P4Info（表/动作清单）
2. 选择主控制器（Master Arbitration）
3. 推送 Pipeline 配置
4. 下发表项（Write）
5. 收发报文（Packet-in/Packet-out）
```

---

## 三、P4Info 文件

编译 P4 程序时会生成 `.p4i` 文件，描述：

- 哪些 Table
- 哪些 Action
- 哪些 Match 字段

控制器通过 P4Info 了解如何与交换机通信。

---

## 四、Python 控制平面示例

```python
import p4runtime_lib.bmv2

# 连接 BMv2
conn = Bmv2Connection(
    device_id=0,
    grpc_addr='localhost:50051',
    election_id=(0, 1)
)
conn.setup()

# 推送 Pipeline
conn.push_pipeline_config(
    p4info='build/hello.p4i',
    bmv2_json='build/hello.json'
)

# 下发表项
entry = conn.table_entry(
    table_name='MyIngress.forward',
    action_name='MyIngress.set_egress',
    match={'hdr.ethernet.dstAddr': encode('00:00:00:00:00:01', 48)},
    action_params={'port': encode(1, 9)}
)
conn.write_table_entry(entry)
```

## 五、Packet-in / Packet-out

### Packet-in

交换机把报文发给控制器：

```p4
action send_to_controller() {
    // 标记发送给控制器
}
```

### Packet-out

控制器向交换机发送报文：

```python
conn.packet_out(packet)
```

---

## 六、小结

| 概念 | 作用 |
|------|------|
| P4Runtime | 控制平面-数据平面通信协议 |
| P4Info | 表/动作的元数据描述 |
| Packet-in | 报文从交换机发给控制器 |
| Packet-out | 报文从控制器发给交换机 |
