L2 交换机只在同一个子网内转发。这一篇实现一个 IPv4 路由器，支持跨子网转发。

> 本文基于 [P4 中文教程](https://github.com/zhh2001/p4-language-guide-zh) examples/03-ipv4-router 整理。

---

## 一、实验目标

实现一个 IPv4 路由器：

- 根据目的 IP 地址最长前缀匹配（LPM）
- 修改源/目的 MAC 地址
- TTL 递减
- 重新计算校验和

---

## 二、P4 程序分析

### 2.1 核心 table

```p4
table ipv4_lpm {
    key = {
        hdr.ipv4.dstAddr: lpm;
    }
    actions = {
        set_nhop;
        drop;
    }
    size = 1024;
    default_action = drop();
}
```

### 2.2 动作定义

```p4
action set_nhop(bit<9> port, bit<32> nhop_ipv4) {
    standard_metadata.egress_spec = port;
    hdr.ipv4.dstAddr = nhop_ipv4;
    hdr.ipv4.ttl = hdr.ipv4.ttl - 1;
}

action rewrite_mac(bit<48> src_mac, bit<48> dst_mac) {
    hdr.ethernet.srcAddr = src_mac;
    hdr.ethernet.dstAddr = dst_mac;
}
```

### 2.3 apply 逻辑

```p4
apply {
    // 检查 TTL
    if (hdr.ipv4.isValid()) {
        if (hdr.ipv4.ttl > 1) {
            ipv4_lpm.apply();       // 路由查找
            rewrite_table.apply();  // MAC 改写
        } else {
            mark_to_drop(standard_metadata);  // TTL 过期，丢弃
        }
    }
}
```

## 三、路由表配置

### 3.1 添加路由表项

```
s1> table_add ipv4_lpm set_nhop 10.0.1.0/24 => 1 10.0.1.1
s1> table_add ipv4_lpm set_nhop 10.0.2.0/24 => 2 10.0.2.1
```

### 3.2 添加 MAC 改写表项

```
s1> table_add rewrite_table rewrite_mac 10.0.1.1 => 00:00:00:00:01:01 00:00:00:00:01:02
s1> table_add rewrite_table rewrite_mac 10.0.2.1 => 00:00:00:00:02:01 00:00:00:00:02:02
```

## 五、校验和更新

IPv4 校验和只覆盖 IPv4 头部。每次修改头部字段（如 TTL），都需要重新计算。

### 5.1 BMv2 自动计算

如果在 `ComputeChecksum` 中声明了 IPv4 校验和字段，BMv2 会自动计算：

```p4
control ComputeChecksum(inout Headers hdr, inout Meta meta) {
    apply {
        verify_checksum(
            true,
            { hdr.ipv4.version, hdr.ipv4.ihl, hdr.ipv4.diffserv,
              hdr.ipv4.totalLen, hdr.ipv4.identification, hdr.ipv4.flags,
              hdr.ipv4.fragOffset, hdr.ipv4.ttl, hdr.ipv4.protocol,
              hdr.ipv4.srcAddr, hdr.ipv4.dstAddr },
            hdr.ipv4.hdrChecksum,
            HashAlgorithm.csum16
        );
    }
}
```

### 5.2 常见校验和错误

如果 ping 不通，检查：
- 校验和是否正确更新
- TTL 是否递减
- MAC 地址是否改写

```python
from scapy.all *

# 构造跨子网 ICMP 包
pkt = Ether(src="00:00:00:00:01:02", dst="00:00:00:00:01:01") / \
      IP(src="10.0.1.2", dst="10.0.2.2", ttl=64) / \
      ICMP()

sendp(pkt, iface="h1-eth0")
```

## 七、V1Model 六大块的协作

在这个实验中，V1Model 的六大块分工明确：

| 块 | 作用 |
|----|------|
| Parser | 解析 Ethernet + IPv4 |
| VerifyChecksum | 验证 IPv4 校验和 |
| MyIngress | 路由查找、TTL 递减、MAC 改写 |
| MyEgress | 通常留空 |
| ComputeChecksum | 重新计算 IPv4 校验和 |
| Deparser | 输出 Ethernet + IPv4 |
