## 一、PSA：可移植架构

PSA（Portable Switch Architecture）是 P4 语言官方定义的可移植架构，目标是让 P4 程序能在不同硬件上运行。

### 1.1 PSA vs V1Model

| 特性 | V1Model | PSA |
|------|---------|-----|
| 定位 | 参考实现 | 生产标准 |
| 适用硬件 | BMv2 | 商业交换机 |
| 功能完整度 | 基础 | 完整 |
| 学习曲线 | 低 | 高 |

### 1.2 PSA 的流水线

```
Parser → Ingress → Traffic Manager → Egress → Deparser
```

PSA 增加了 Traffic Manager（流量管理器），支持：
- 队列调度
- 流量整形
- 报文克隆
- 缓冲管理

### 1.3 什么时候用 PSA

- 需要生产级功能（QoS、镜像、缓冲）
- 目标硬件支持 PSA
- 需要跨平台移植

## 二、TNA：Tofino 原生架构

TNA（Tofino Native Architecture）是 Barefoot/Intel Tofino 交换芯片的专用架构。

### 2.1 TNA 的特点

- 针对 Tofino 硬件优化
- 支持更复杂的流水线
- 有更多的Extern对象
- 性能更高

### 2.2 TNA vs PSA

| 特性 | PSA | TNA |
|------|-----|-----|
| 可移植性 | 高 | 低（Tofino 专用） |
| 功能丰富度 | 中 | 高 |
| 性能 | 一般 | 优化 |

### 2.3 TNA 的学习建议

初学阶段不需要深入 TNA，等熟悉 P4 基础后再根据需要学习。

## 三、INT：带内网络遥测

INT（In-band Network Telemetry）是 P4 最热门的研究方向之一，利用数据平面实时采集网络状态。

### 3.1 INT 是什么

传统网络监控（如 SNMP）是"外带"的——控制平面主动查询。INT 是"内带"的——报文在转发过程中自动携带网络状态信息。

```
传统监控：交换机 → (查询) → 控制器 → (统计) → 管理系统
INT 监控：报文 → (携带状态) → 收集器 → (分析) → 管理系统
```

### 3.2 INT 能采集什么

| 指标 | 说明 |
|------|------|
| 时延 | 报文在每个节点的处理时间 |
| 队列深度 | 交换机队列的占用情况 |
| 跳数 | 报文经过的交换机数量 |
| 端口 ID | 报文进出的端口号 |
| 时间戳 | 精确的纳秒级时间戳 |

### 3.3 INT 的工作原理

1. **源节点**：在报文中插入 INT Header
2. **中间节点**：每个交换机把自己的状态信息追加到报文
3. **目的节点**：移除 INT Header，将信息发送给收集器
4. **分析**：收集器分析网络状态

### 3.4 INT 的应用场景

- **网络故障定位**：快速发现丢包、延迟的根因
- **负载均衡**：根据实时队列状态调整流量分配
- **网络安全**：检测异常流量模式
- **性能优化**：监控链路利用率

### 3.5 参考项目

- [TT-INT](https://github.com/zhh2001/TT-INT) — 张恒华师兄的 INT 实现
- [INT-MN](https://github.com/p4lang/p4-apps/tree/master/int) — 官方 INT 示例

## 四、网络内计算（In-Network Computing）

利用交换机的硬件转发能力，在数据平面完成计算任务。

### 4.1 典型应用

| 应用 | 说明 |
|------|------|
| 聚合 | 在交换机内完成 MapReduce 类似的聚合 |
| 缓存 | 利用交换机 SRAM 做热门内容缓存 |
| AI 推理 | 在交换机内做简单的神经网络推理 |
| 加密 | 在数据平面完成加解密 |

### 4.2 优势

- **超低延迟**：无需绕道服务器
- **高吞吐**：利用硬件并行处理
- **节省带宽**：减少到服务器的流量

## 五、P4 安全应用

在数据平面实现安全功能，线速防护。

### 5.1 DDoS 防护

```p4
// 简单的速率限制
Counter<bit<64>, bit<32>>(1024, CounterType.PACKETS) flow_counter;

table rate_limit {
    key = { hdr.ipv4.srcAddr: lpm; }
    actions = { drop; NoAction; }
    counters = flow_counter;
}
```

### 5.2 入侵检测

- 在 P4 中实现简单的特征匹配
- 结合控制器做复杂分析
- 实时阻断攻击流量

### 5.3 加密流量审计

- 检测 TLS 握手特征
- 统计加密流量比例
- 识别异常加密模式
