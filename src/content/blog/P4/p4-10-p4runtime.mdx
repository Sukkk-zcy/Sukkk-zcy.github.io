## 一、P4Runtime 是什么

P4Runtime 是一个标准的控制平面-数据平面通信协议，定义了：

1. **P4Info** — 表/动作/匹配类型的元数据
2. **gRPC 传输** — 控制器和交换机之间的通信
3. **StreamChannel** — 双向流，用于 Packet-in/Packet-out

### 1.1 为什么用 P4Runtime

- 比 Thrift CLI 更灵活
- 支持 Packet-in/Packet-out（报文发送给控制器）
- 标准化，可移植

## 三、P4Runtime 工作流程

```
1. 控制器读取 P4Info（表/动作清单）
2. 选择主控制器（Master arbitration）
3. 推送 pipeline 配置
4. 读写表项
5. 收发 Packet-in/Packet-out
```

## 四、Python 控制平面示例

### 4.1 基本连接

```python
import p4runtime_lib.bmv2
from p4runtime_lib.error_utils import handle_grpc_error

# 创建连接
bmv2_connection = p4runtime_lib.bmv2.Bmv2Connection(
    device_id=0,
    grpc_addr='localhost:50051',
    election_id=(0, 1),  # (high, low)
    proto_filenode='build/l2_switch.p4i'
)

# 启动连接
bmv2_connection.setup()
```

### 4.2 下发表项

```python
from p4runtime_lib.convert import encode

# 添加 MAC 表项
table_entry = bmv2_connection.table_entry(
    table_name='MyIngress.l2_table',
    action_name='MyIngress.set_egress',
    match={
        'hdr.ethernet.dstAddr': encode('00:00:00:00:00:01', 48)
    },
    action_params={
        'port': encode(1, 9)
    }
)

bmv2_connection.write_table_entry(table_entry)
```

### 4.3 读取表项

```python
entries = bmv2_connection.read_table_entries(
    table_name='MyIngress.l2_table'
)

for entry in entries:
    print(entry)
```

## 五、Packet-in / Packet-out

### 5.1 什么是 Packet-in

当 P4 程序中调用 `send_to_controller()` 时，报文会被发送给控制器，这就是 Packet-in。

### 5.2 什么是 Packet-out

控制器可以向交换机发送报文，这就是 Packet-out。

### 5.3 L2 学习交换机示例

```p4
// P4 程序中
action send_to_controller() {
    meta.digest_data = hdr.ethernet.srcAddr;
}

table learning_table {
    key = { hdr.ethernet.srcAddr: lpm; }
    actions = { send_to_controller; set_egress; }
    default_action = send_to_controller();  // 未学习的 MAC 发给控制器
}
```

## 六、完整流程

### 6.1 控制器启动流程

```python
# 1. 连接 BMv2
conn = Bmv2Connection(device_id=0, grpc_addr='localhost:50051')

# 2. 选择主控制器
conn.setup()

# 3. 推送 pipeline
conn.push_pipeline_config(
    p4info='build/l2_switch.p4i',
    bmv2_json='build/l2_switch.json'
)

# 4. 下发表项
conn.write_table_entry(...)

# 5. 运行事件循环
conn.grpc_loop()
```

### 6.2 完整示例

```python
#!/usr/bin/env python3
from p4runtime_lib.bmv2 import Bmv2Connection
from p4runtime_lib.convert import encode

def main():
    # 连接
    conn = Bmv2Connection(
        device_id=0,
        grpc_addr='localhost:50051',
        election_id=(0, 1),
        proto_filenode='build/l2_switch.p4i'
    )
    conn.setup()

    # 推送配置
    conn.push_pipeline_config(
        p4info='build/l2_switch.p4i',
        bmv2_json='build/l2_switch.json'
    )

    # 下发 MAC 表项
    for mac, port in [('00:00:00:00:00:01', 1), ('00:00:00:00:00:02', 2)]:
        entry = conn.table_entry(
            table_name='MyIngress.l2_table',
            action_name='MyIngress.set_egress',
            match={'hdr.ethernet.dstAddr': encode(mac, 48)},
            action_params={'port': encode(port, 9)}
        )
        conn.write_table_entry(entry)

    print("表项下发完成")

if __name__ == '__main__':
    main()
```

## 七、常见问题

### 7.1 gRPC 连接失败

检查：
- BMv2 是否以 `--grpc-server-addr` 启动
- 端口是否正确（默认 50051）
- 防火墙是否放行

### 7.2 表项下发失败

检查：
- P4Info 文件路径是否正确
- 表名/动作名是否匹配
- match 字段类型是否正确

### 7.3 Packet-in 收不到

检查：
- P4 程序是否调用了 `send_to_controller()`
- 是否配置了 CPU port
- 是否注册了 packet_in_handler
