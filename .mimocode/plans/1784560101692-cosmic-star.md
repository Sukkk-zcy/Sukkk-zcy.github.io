## 新文章大纲

### p4-01: P4 是什么？为什么需要它
**定位**：概念入门，解释 P4 的背景和价值

内容：
- 传统网络设备的局限性
- OpenFlow 的贡献与不足
- P4 的诞生：让数据面可编程
- P4 的三大设计目标：协议无关、架构无关、可移植
- P4 能做什么：自定义协议、网络遥测、安全检测
- 配图建议：网络设备演进图、P4 vs OpenFlow 对比图

### p4-02: P4 程序的骨架：从输入到输出
**定位**：理解 P4 程序的整体结构

内容：
- 一个 P4 程序长什么样
- Parser（解析器）：从字节流提取报头
- Control（控制逻辑）：匹配-动作处理
- Deparser（反解析器）：把报头重新打包
- 五元组示例：Ethernet → VLAN → IPv4 → TCP → Payload
- 配图建议：P4 程序结构图、数据包处理流程图

### p4-03: Header：网络报文的"身份证"
**定位**：理解 header 类型的概念

内容：
- 什么是 header？为什么需要它
- header 的定义方式（类似 C 结构体）
- header 的基本字段类型：bit<W>、int<W>
- header stack：处理重复报头（如 VLAN 标签）
- header union：处理可选报头
- 实际例子：Ethernet 头、IPv4 头
- 配图建议：Ethernet 帧结构图、header 示例

### p4-04: Parser：报文解析的状态机
**定位**：理解 Parser 的工作原理

内容：
- Parser 本质是一个有限状态机
- start → extract → select → accept/reject
- select 语句：根据字段值跳转
- extract：从报文提取字段
- verify：条件校验
- lookahead：偷看但不消费
- 配图建议：状态机流程图、Parser 示例

### p4-05: Match-Action：网络转发的核心
**定位**：理解匹配-动作表的工作原理

内容：
- 什么是 Match-Action 表
- 三种匹配类型：exact、lpm、ternary
- 动作（Action）：对报文的操作
- 表项：由控制平面下发
- 优先级和默认动作
- 配图建议：Match-Action 流程图、三种匹配对比

### p4-06: Control Block：连接 Parser 和 Table
**定位**：理解控制块的作用

内容：
- Control Block 是程序的"大脑"
- apply 语句：按顺序执行逻辑
- 条件判断、循环
- header 的 isValid() 检查
- 修改 header 字段
- 配图建议：Control Block 结构图

### p4-07: V1Model：BMv2 的参考架构
