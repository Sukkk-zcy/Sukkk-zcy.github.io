## 三、P4 程序分析

### 3.1 核心 table

```p4
table l2_table {
    key = {
        hdr.ethernet.dstAddr: exact;
    }
    actions = {
        set_egress;
        broadcast;
    }
    size = 1024;
    default_action = broadcast();
}
```

### 3.2 动作定义

```p4
action set_egress(bit<9> port) {
    standard_metadata.egress_spec = port;
}

action broadcast() {
    standard_metadata.mcast_grp = 1;  // 组播组 1
}
```

### 3.3 apply 逻辑

```p4
apply {
    l2_table.apply();
}
```

## 四、运行实验

### 4.1 一键运行

```bash
sudo bash run.sh
```

这个脚本会：
1. 编译 P4 程序
2. 启动 BMv2 交换机
3. 启动 Mininet 拓扑（2 交换机 + 4 主机）
4. 配置静态 MAC 表项

### 4.2 验证连通性

```bash
mininet> h1 ping h2
```

如果 MAC 表项已配置，h1 应该能 ping 通 h2。

## 五、手动配置表项

### 5.1 进入 simple_switch_CLI

在另一个终端：

```bash
sudo simple_switch_CLI --thrift-port 9090
```

### 5.2 查看当前表项

```
s1> table_dump l2_table
```

### 5.3 添加表项

```
s1> table_add l2_table set_egress 00:00:00:00:00:01 => 1
s1> table_add l2_table set_egress 00:00:00:00:00:02 => 2
```

### 5.4 删除表项

```
s1> table_delete l2_table 00:00:00:00:00:01
```

## 七、编译过程详解

### 7.1 p4c 编译

```bash
p4c --target bmv2 --arch v1model --p4runtime-files p4.p4i l2_switch.p4 -o build
```

参数说明：
- `--target bmv2`：目标是 BMv2
- `--arch v1model`：使用 V1Model 架构
- `--p4runtime-files`：生成 P4Runtime 信息文件

### 7.2 生成的文件

| 文件 | 作用 |
|------|------|
| `l2_switch.json` | BMv2 配置文件 |
| `l2_switch.p4i` | P4Runtime 信息（表/动作清单） |

## 八、拓扑配置

### 8.1 run.sh 做了什么

```bash
#!/bin/bash
# 编译
p4c --target bmv2 --arch v1model l2_switch.p4 -o build

# 启动交换机
simple_switch build/l2_switch.json --log-console \
  -i 0@veth0 -i 1@veth2 -i 2@veth4 -i 3@veth6 \
  --thrift-port 9090 &

# 等待启动
sleep 2

# 配置表项
simple_switch_CLI --thrift-port 9090 <<EOF
table_add l2_table set_egress 00:00:00:00:00:01 => 1
table_add l2_table set_egress 00:00:00:00:00:02 => 2
EOF

# 启动 Mininet
sudo mn --topo single,2 --host link --switch default,protocols=OpenFlow13 \
  --controller none
```

## 九、常见问题

### 9.1 ping 不通

**检查**：
- MAC 表项是否配置正确
- 端口号是否对应
- 用 `table_dump` 查看表项

### 9.2 编译错误

**常见原因**：
- header 字段名拼写错误
- match_kind 不匹配
- action 参数类型不对

### 9.3 端口号问题

BMv2 的端口号从 0 开始，Mininet 的端口号可能不同。检查 `-i` 参数的映射。
