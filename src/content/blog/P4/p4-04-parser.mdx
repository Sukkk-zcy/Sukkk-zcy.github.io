上一篇理解了 Header，这一篇来看 Parser——它是怎么把字节流"翻译"成结构化报头的。

---

## 一、Parser 本质是什么

Parser 是一个**有限状态机（FSM）**。每个状态做两件事：

1. **Extract**：从报文中提取若干字节到对应的 Header
2. **Select**：根据某个字段的值，决定跳转到下一个状态

整个过程从 `start` 状态开始，到 `accept` 状态结束。


---

## 二、状态机的基本结构

```p4
parser MyParser(packet_in pkt, out Headers hdr) {
    state start {
        pkt.extract(hdr.ethernet);           // 提取以太网头
        transition select(hdr.ethernet.etherType) {
            0x0800: parse_ipv4;              // IPv4 → 继续解析
            default: accept;                 // 其他 → 结束
        }
    }

    state parse_ipv4 {
        pkt.extract(hdr.ipv4);
        transition accept;
    }
}
```

---

## 三、select：分支跳转

`select` 根据字段值选择下一个状态：

```p4
transition select(hdr.ethernet.etherType) {
    0x0800: parse_ipv4;    // IPv4
    0x0806: parse_arp;     // ARP
    0x8100: parse_vlan;    // VLAN
    default: accept;       // 不认识就结束
}
```

### 多字段 select

```p4
transition select(hdr.ipv4.protocol, hdr.ipv4.dstAddr) {
    (0x06, 10.0.0.1): parse_tcp_to_server;
    (0x11, *): parse_udp_any;  // * 表示任意值
    default: accept;
}
```

---

## 四、extract：提取报头

```p4
pkt.extract(hdr.ethernet);  // 提取 14 字节（以太网头）
pkt.extract(hdr.ipv4);      // 提取 20 字节（IPv4 头）
```

### 带长度的 extract

IPv4 头长度可变（有选项字段）：

```p4
pkt.extract(hdr.ipv4, (bit<32>)(hdr.ipv4.ihl * 32));
```

---

## 五、verify：条件校验

```p4
verify(hdr.ipv4.version == 4, error.Ipv4IncorrectVersion);
verify(hdr.ipv4.ihl >= 5, error.Ipv4TooShort);
```

不满足条件会跳转到 `reject`。

---

## 六、lookahead：偷看但不消费

```p4
bit<16> nextType = lookahead(bit<16>);
transition select(nextType) {
    0x0800: parse_ipv4;
    default: accept;
}
```

不移动解析指针，只"偷看"下一个字段。

---

## 七、完整示例：Ethernet + IPv4

```p4
parser MyParser(packet_in pkt, out Headers hdr) {
    state start {
        pkt.extract(hdr.ethernet);
        transition select(hdr.ethernet.etherType) {
            0x0800: parse_ipv4;
            default: accept;
        }
    }

    state parse_ipv4 {
        pkt.extract(hdr.ipv4);
        verify(hdr.ipv4.version == 4, error.Ipv4IncorrectVersion);
        transition accept;
    }
}
```

---

## 八、常见踩坑

| 问题 | 原因 |
|------|------|
| 编译报错"未定义的状态" | select 中引用的状态没有定义 |
| 运行时崩溃 | 忘记 extract 某个 header |
| Parser 无限循环 | 状态跳转逻辑有误 |

---

## 九、小结

| 概念 | 作用 |
|------|------|
| `state` | 状态机的一个节点 |
| `extract` | 从报文提取字节到 header |
| `select` | 根据字段值选择下一个状态 |
| `verify` | 条件校验 |
| `lookahead` | 偷看下一个字段 |
| `accept` | 解析成功 |
| `reject` | 解析失败 |
