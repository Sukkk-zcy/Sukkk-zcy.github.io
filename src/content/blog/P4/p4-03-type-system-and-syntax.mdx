## 一、P4 的类型长什么样

P4 的类型系统和 C 差不多，但有几个关键区别。

### 1.1 基本整数类型

P4 里最常用的类型是 `bit<W>`，表示 W 位无符号整数：

```p4
bit<8>   port;      // 8 位，0~255
bit<16>  etherType; // 16 位，0~65535
bit<48>  macAddr;   // 48 位，MAC 地址
bit<32>  ipv4Addr;  // 32 位，IPv4 地址
```

也有有符号的 `int<W>`，但网络报文处理中很少用到。

### 1.2 整数字面量：宽度前缀

P4 的字面量可以显式指定宽度和进制，格式是 `<宽度>w<值>`：

```p4
32w255           // bit<32>，值 255
32w0xFF          // 同上，十六进制
8w0b10101010     // bit<8>，值 170
16w0x0800        // bit<16>，值 2048（IPv4 的 etherType）
```

数字之间可以用 `_` 分隔提高可读性：

```p4
48w0x00_11_22_33_44_55    // 以太网 MAC 地址
```

> **踩坑记录**：P4 的 `0377` 是**十进制 377**，不是八进制！C 里 `0377` 等于 255，但 P4 不认八进制前缀。想表达八进制请写 `0o377`。我第一次就被这个坑了，编译不报错但值不对。

### 1.3 布尔和其他类型

```p4
bool flag = true;   // 布尔值
error NoError;      // 错误类型（parser 里用）
```

P4 **没有浮点数**，**没有指针**，**没有字符串变量**（只有字面量）。这是硬件约束——交换机芯片不支持这些。

## 二、header：P4 最重要的类型

`header` 是 P4 对**网络报头**的抽象。如果说 P4 是"描述报文"的语言，那 header 就是"描述一个报头"的核心工具。

### 2.1 定义一个 header

用 Ethernet 帧举例：

```p4
header Ethernet_h {
    bit<48> dstAddr;    // 目的 MAC
    bit<48> srcAddr;    // 源 MAC
    bit<16> etherType;  // 类型字段
}
```

这就像 C 的 struct，但它多了一个**隐藏的有效位（validity bit）**。

### 2.2 有效位：P4 最常见的 bug 来源

每个 header 变量都有一个有效位，表示"这个报头在报文中是否真的存在"：

```p4
Ethernet_h eth;
// eth.isValid() == false，有效位是 false

eth.dstAddr = 0x00_11_22_33_44_55;
// 可以赋值，但 eth.isValid() 仍然是 false！

eth.setValid();
// 现在 eth.isValid() == true
```

> **踩坑记录**：**忘记 `setValid()` 是 P4 最高频的 bug**。我跑 Hello P4 的时候，编译通过、运行不报错，但就是收不到包。查了半天发现是 header 没有 setValid，`emit` 不会输出无效的 header。记住：**写了 header 就要 setValid**。

反过来，删除表项的时候要用 `setInvalid()`：

```p4
eth.setInvalid();  // 有效位变 false
```

### 2.3 header 的比较

```p4
h1 == h2   // 类型相同 + 有效位相同 + 所有字段相等
h == {#}   // 等价于 !h.isValid()
```

`{#}` 表示一个"无效的 header"，常用于初始化。

### 2.4 header 的初始化

```p4
header H { bit<32> x; bit<32> y; }
H h;
h = { 10, 12 };          // 按字段顺序
h = { y = 12, x = 10 };  // 按名称
h = {#};                 // 无效报头
```

## 三、header stack：定长数组

网络协议经常有"同一类型的报头重复多次"的情况，比如 MPLS 标签栈、VLAN 嵌套。P4 用 **header stack** 来表示：

```p4
header Mpls_h {
    bit<20> label;
    bit<3>  tc;
    bit<1>  bos;
    bit<8>  ttl;
}

Mpls_h[10] mpls;   // 最多 10 层 MPLS 标签
```

常用操作：

| 表达式 | 含义 |
|--------|------|
| `mpls[0]` | 下标访问第 0 个 |
| `mpls.size` | 栈大小（常量 10） |
| `mpls.next` | 下一个待解析的（parser 里用） |
| `mpls.last` | 最后一个已解析的（parser 里用） |
| `mpls.push_front(n)` | 前面插入 n 个空位 |
| `mpls.pop_front(n)` | 前面移除 n 个 |

> **注意**：`next` 和 `last` **只能在 parser 里用**，control 块里不能用。

## 四、header_union：同一时刻只能有一个

有时候同一段数据可能是几种报头中的任意一种。比如一个报文的网络层可能是 IPv4 也可能是 IPv6：

```p4
header_union IP_h {
    IPv4_h v4;
    IPv6_h v6;
}
```

关键规则：**给某个成员 setValid() 会自动把其他成员 invalid**。

```p4
ip.v4.setValid();   // v4 有效，v6 自动无效
ip.v6.setValid();   // v6 有效，v4 自动无效
```

> **踩坑记录**：我一开始以为可以同时有两个 valid 的成员，结果发现 header_union 的 `isValid()` 只在"恰好一个成员 valid"时返回 true。这和 C 的 union 完全不同，别搞混了。

---

## 五、struct：纯数据容器

struct 和 C 的 struct 一样——没有有效位，所有字段都是普通数据：

```p4
struct metadata_t {
    bit<9>  ingress_port;
    bit<16> l4_src_port;
    bool    is_multicast;
}
```

典型用途：
- **headers 结构体**：把所有解析出来的报头打包在一起
- **metadata 结构体**：传递流水线中间的元数据

## 六、语法：和 C 的关键区别

### 6.1 switch 不自动 fall-through

这是和 C **完全相反**的行为：

```p4
switch (x) {
    1:       // 没有 { }，自动 fall-through 到 2
    2: { /* 1 或 2 都走这里 */ }
    3: { /* 仅 3，执行完跳出 switch */ }
}
```

> **踩坑记录**：C 里 switch 是"不 break 就 fall-through"，P4 是"没有 { } 才 fall-through"。我写代码的时候习惯性地没加 { }，结果逻辑完全反了。

### 6.2 参数方向：必须声明

P4 函数的每个参数都必须声明方向：

```p4
bit<32> max(in bit<32> a, in bit<32> b) {
    return (a > b) ? a : b;
}
```

| 方向 | 含义 | 类比 |
|------|------|------|
| `in` | 只读输入 | C++ `const T&` |
| `out` | 必须赋值的输出 | C 的出参 |
| `inout` | 可读可改 | C++ `T&` |
| 无方向 | 编译期常量 | template 参数 |

只有 `in` 和无方向参数可以有默认值。

### 6.3 不支持递归

P4 **不支持任何形式的递归函数**——直接递归和间接递归都不行。这是硬件约束：交换机芯片需要静态展开所有逻辑，递归意味着不确定的深度。

### 6.4 没有 ++、没有 +=、没有浮点

```p4
// 以下全部不合法：
x++;           // ❌ 不支持自增
x += 1;        // ❌ 不支持复合赋值
float f = 1.0; // ❌ 没有浮点类型

// 正确写法：
x = x + 1;     // ✅
```

### 6.5 位切片和拼接

```p4
bit<32> ip;
bit<8> first_byte = ip[31:24];   // 取高位字节（闭区间，高位在前）

bit<16> a = 0xAA00;
bit<8>  b = 0xFF;
bit<24> c = a ++ b;              // 0xAA00FF（拼接）
```

### 6.6 类型转换要显式

P4 的隐式转换极少，大多数情况需要你显式写转换：

```p4
bit<8>  x;
bit<16> y;
bit<16> z = (bit<16>)x + y;    // ✅ 显式转换
// bit<16> z = x + y;          // ❌ 编译报错：位宽不同
```

> **踩坑记录**：P4 故意不帮你做隐式转换，就是为了避免 C 那种"无声的 bug"。刚开始觉得麻烦，但习惯之后会觉得编译器帮你检查类型是好事。

## 七、类型总结对照表

| 类型 | 用途 | 有有效位？ |
|------|------|-----------|
| `bit<W>` | 无符号整数 | 否 |
| `int<W>` | 有符号整数 | 否 |
| `bool` | 布尔 | 否 |
| `header` | 网络报头 | **是** |
| `header stack` | 报头数组 | 是（每个元素） |
| `header_union` | 互斥报头组 | 是（成员级） |
| `struct` | 数据容器 | 否 |
| `enum` | 枚举 | 否 |
| `error` | 错误码 | 否 |
| `varbit<W>` | 可变长度位串 | 是 |
