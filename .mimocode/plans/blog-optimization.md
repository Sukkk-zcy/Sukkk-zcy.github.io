# 博客优化方案

## 问题分析

### 1. Git 发布需要多次授权
**原因**：每次 git add、commit、push 都触发权限确认
**解决方案**：
- 已配置 `credential.helper=store`
- remote 使用 SSH URL（不需要授权）
- 需要检查为什么仍然需要授权

### 2. 构建速度慢
**原因**：
- Astro 构建需要处理大量资源
- `npm run build` 包含 check 步骤
- 图片优化（sharp）耗时
**解决方案**：
- 跳过不必要的 check
- 使用增量构建
- 优化图片处理

### 3. 部署流程复杂
**原因**：每次都要初始化 git、复制文件、推送
**解决方案**：
- 创建一键部署脚本
- 使用 gh-pages 分支而非每次重建

## 优化计划

### Phase 1: 修复 Git 授权问题
1. 检查 SSH key 配置
2. 确保 remote 使用 SSH
3. 测试 git push 是否需要授权

### Phase 2: 优化构建速度
1. 简化 build 命令（跳过 check）
2. 添加增量构建支持
3. 优化图片处理配置

### Phase 3: 优化部署流程
1. 创建 `deploy.ps1` 一键脚本
2. 添加缓存机制
3. 简化 gh-pages 更新流程

### Phase 4: 文章检查
1. 检查所有 P4 文章格式
2. 确保参考文献完整
3. 验证链接有效性

## 具体修改
